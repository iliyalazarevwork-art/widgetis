# Идеи виджетов для Horoshop

Список реальных болей магазинов на Horoshop (2022–2026), которые можно закрыть отдельными виджетами в Widgetis. Каждый пункт — независимый виджет с понятной ценностью и готовностью платить.

Источники: фидбек владельцев магазинов в чатах Horoshop, тикеты в поддержку, цитаты из переписок.

---

## 🔥 Killer-features (высокая готовность платить)

### 1. SMS/Viber-OTP подтверждение телефона + антифрод
**Проблема (2022 → 2026):** через FB/Google льются арбитражные фейк-лиды (10–30% всех заказов). Кнопка «заблокировать клиента» в Хорошопе не работает. Магазины пишут кустарные SMS-проверки на JS, платят за NP-возвраты, обучают рекламные алгоритмы на мусоре.

> [2024-03-19] Denys Pavlov: «9 сайтів на Хорошоп, на кожен з сайтів кожен день йдуть фейки»
> [2025-10-09] Yevhen: «боремось вже третій рік… єдине що допомогло — перевірка номера через смс»

**Виджет:** OTP в чекауте + база подозрительных номеров + скоринг по UTM/IP/fingerprint + автопометка в CRM.

---

### 2. Uptime-monitor + автопауза рекламы при падении сайта
**Проблема (2022 → 2026, 10+ крупных инцидентов):** сайт лежит, реклама в FB/Google продолжает сжигать бюджет. Хорошоп не уведомляет о сбоях. Магазины теряют десятки тысяч за инцидент.

> [2024-09-12] Galeev Denis: «коли лягають проєкти треба своєчасно вимикати рекламу, в деяких великі бюджети»
> [2026-03-26] Vladimir: «Підтримки нуль. Ніхто не попередив про проблеми»

**Виджет:** мониторинг uptime + auto-pause Google Ads / Facebook Ads через API + SMS/Telegram алерт владельцу.

---

### 3. Авто-бэкапы магазина + восстановление одной кнопкой
**Проблема (2024):** реальный кейс взлома и удаления товаров. Восстанавливать из ничего.

**Виджет:** внешний сервис автобэкапов через API Horoshop + UI восстановления.

---

## 💬 Отзывы и UGC

### 4. Фото/видео отзывы + модерация + auto-request
**Проблема (2022 → 2026):** Хорошоп позволяет только текст и ссылку — никто ссылками не пользуется. Нет: экспорта, массовой модерации, авто-запроса после доставки, агрегации с Розетки/Епіцентра, «звёздочек» в форме.

> [2024-11-13] Руслан Лелет: «писав рік назад, ще не реалізовано? Напишу через рік»
> [2025-08-07] Disabl: «Як вигрузити відгуки? Парсимо Screaming Frog'ом по своєму сайту»

**Виджет:** UGC-галерея фото/видео + email-триггер «оставьте отзыв за купон» + модерация + Google Reviews fid. *Уже делается в `module-photo-reviews`.*

## 🛒 Чекаут и конверсия

### 5. Маска телефона по странам + B2B-поля (NIP/VAT/Company)
**Проблема (2022 → 2026):** маска есть только для UA. Польские B2B-покупатели (~30% продаж в EU) не могут получить фактуру с NIP. «Хорошоп не готов для международной продажі».

**Виджет:** чекаут-overlay с гео-зависимыми масками, NIP/VAT/Company, валидацией.

---

### 6. Поиск отделений InPost / Нова Пошта по карте + индексу
**Проблема (2022 → 2026):** в Хорошопе — выпадающий список из тысяч отделений. Поляки привыкли к карте + индексу. Конверсия в EU режется.

**Виджет:** умный автокомплит отделений с картой и поиском по индексу.

---

### 7. Гео-popup выбора языка/страны при первом входе
**Проблема (2022 → 2025):** нет авто-определения страны.

**Виджет:** Geo-popup с auto-detect через GeoIP, страна/язык по умолчанию.

---

### 8. Callback / «Перезвоните мне»
**Проблема (2022 → 2024):** «Третина клієнтів лінується в корзину класти — просто дзвінок замовляє». Особенно для EU, где платят при доставке.

**Виджет:** callback с триггерами (exit-intent, time-on-page).

---

### 9. Расчёт доставки по весу/габаритам + ограничение сервиса по товару
**Проблема (2022 → 2026):** «Вже 2 роки просимо…». Нельзя ограничить почтомат для удочки 130см или Укрпочту для габарита >120см. Менеджеры тратят часы на перезвоны.

**Виджет:** Shipping rules — правила по весу / габаритам / категории / бренду.

---

### 10. Multi-warehouse stock display
**Проблема (2022 → 2024):** «есть в Киеве, нет в Польше».

**Виджет:** показ остатков по складам/городам с картой и адресами.

---

### 11. Back-in-stock notifications
**Проблема (2024 → 2026):** «Сообщить, когда появится».

**Виджет:** подписка на email/Telegram-уведомление о появлении товара.

---

## 💰 Промо и AOV

### 12. Promo-engine: бандлы, 1+1=3, gift with purchase, gift wrap
**Проблема (2022 → 2026):** «Просим с 2019 года». Стандартный «вместе дешевле» не закрывает кейс. Нет gift wrap, нет «купи N — получи скидку», нельзя суммировать скидки.

**Виджет:** правила BOGO / tiered / gift-with-purchase / add-on прямо в корзине.

---

### 13. Гейміфікація: колесо фортуны, бейджи, прогресс-бары
**Проблема (2023 → 2024):** хочется механик удержания.

**Виджет:** колесо фортуны / popup за email / прогресс-бары лояльности.

---

### 14. FOMO-счётчики: «N человек смотрят», «осталось N шт», countdown
**Проблема (2024 → 2026):** триггеры срочности.

**Виджет:** scarcity counter + countdown + real-time viewers.

---

### 15. A/B тесты карточки / фото / цены
**Проблема (2024):** нет встроенного сплит-теста.

**Виджет:** сплит-трафик на варианты карточки.

## 🤖 AI и контент

### 16. AI-описания товаров + AI-перевод каталога
**Проблема (2023 → 2026):** ручное заполнение тысяч карточек невозможно. Делают через ChatGPT в Excel. Конкурент — `digitalsok.agency`.

**Виджет:** генерация описаний (GPT) + DeepL/GPT-перевод с native-проверкой + JSON-LD.

---

### 17. AI-улучшение фото товаров (фон, инфографика, lifestyle)
**Проблема (2026):** растущая ниша. Конкурент — Uplify Visual.

**Виджет:** pipeline через GPT-image / SDXL / коммерческие API.

---

### 18. Морфологический поиск по сайту
**Проблема (с 2020):** конкурент — Algolia.

**Виджет:** instant-search с морфологией + опечатки + синонимы + превью карточек.

## Видалені з каталогу (без реального модуля)

Ці ідеї були у `frontend/src/data/widgets.ts` як задумані виджети, але реального коду у `widget-builder/modules/` немає. Видалено зі сторінки `/widgets`, щоб показувати тільки те, що реально існує.

### purchase-counter — Лічильник покупок
- **Ідея:** Показує скільки людей вже купили цей товар. «127 людей купили цей товар за 24 години».
- **Відмінність від `module-social-proof`:** social-proof показує динамічний лічильник на сторінці товару, але без реальних даних із CRM.
- **Тег:** social-proof · advanced · 3000 грн

### snow — Віджет снігу
- **Ідея:** Ефект падаючого снігу на сайті. Святкова атмосфера на Новий рік/Різдво.
- **Тег:** visual · simple · 1200 грн

### live-viewers — Хто зараз дивиться
- **Ідея:** «14 людей дивляться зараз» на сторінці товару — FOMO у реальному часі.
- **Відмінність від `module-social-proof`:** social-proof показує «купили», а цей — «онлайн-глядачі».
- **Тег:** urgency · advanced · 3000 грн

### cashback — Кешбек-калькулятор
- **Ідея:** Показує суму кешбеку за покупку. Стимул до рішення та повторних покупок.
- **Потребує:** бекенд-інтеграцію з системою нарахування балів.
- **Тег:** loyalty · complex · 6000 грн

### bonus — Бонусна система
- **Ідея:** Повноцінна бонусна програма — нараховує бали за покупки, які можна витратити.
- **Складніша версія cashback** — окремий баланс, витрата на наступне замовлення.
- **Тег:** loyalty · complex · 6000 грн

### countdown — Зворотний відлік
- **Ідея:** Таймер до кінця акції/розпродажу. «До кінця знижки: 09:47:22».
- **Відмінність від `module-exit-intent-popup`:** не прив'язаний до поведінки юзера, а до дати акції.
- **Тег:** urgency · advanced · 3000 грн

### recent-purchase — Хтось щойно купив
- **Ідея:** Pop-up у куті: «Олена з Харкова щойно купила цей товар». FOMO-ефект.
- **Відмінність від `module-social-proof`:** тут конкретна персона і «щойно», а не агрегований лічильник.
- **Тег:** social-proof · advanced · 3000 грн

### quiz — Квіз-рекомендатор
- **Ідея:** 3-5 питань → рекомендація ідеального товару. Конверсія до 25-40%.
- **Відмінність від `module-cart-recommender`:** cart-recommender — карусель у кошику; quiz — окремий flow вибору.
- **Тег:** conversion · complex · 6000 грн
